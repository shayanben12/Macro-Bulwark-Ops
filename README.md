:star2: __**Guia de Utilização: Sistema Multi-Instância (Somente caso queira farmar em varias contas ao mesmo tempo)**__ :star2:
A função multi-instância permite que o script controle vários emuladores simultaneamente, caso você deseje farmar em varias contas ao mesmo tempo.

__**Como Criar e Configurar Novas Instâncias (Clonagem)**__
Para rodar múltiplas contas, você não precisa instalar o jogo e configurar tudo do zero em cada novo emulador. O método correto e mais eficiente é clonar a sua instância principal.

__**Passo a Passo para Clonagem no MuMu Player:**__
**1. Encerre os Processos**Feche completamente o MuMu Player e qualquer jogo que esteja rodando. A clonagem não funciona corretamente com o emulador em uso.

**2. Abra o Gerenciador**No Windows, procure e abra o aplicativo MuMu Multi-Drive (Gerenciador de Múltiplas Instâncias), que é instalado junto com o emulador.

**3. Inicie a Clonagem**Na lista do gerenciador, localize o seu emulador principal (que já possui o jogo instalado e configurado). Clique no botão de Copiar/Clonar localizado ao lado direito da linha correspondente.

**4. Renomeie a Instância**Após o término da cópia, uma nova instância aparecerá na lista. Clique no ícone de edição (lápis) e altere o nome para algo objetivo, como Conta Primaria, Conta Secundaria, etc.
Atenção: O script lê exatamente este nome para exibir no terminal e salvar o seu progresso.

**5. Execução**Clique em iniciar nas instâncias que deseja rodar, abra o jogo em cada uma, faça o login nas contas correspondentes e inicie o script.

__**Como iniciar o sistema Multi-Instância:**__
**1. Preparação dos Emuladores**Abra todas as instâncias do MuMu Player que você planeja utilizar na sessão.

**2. Habilitação do ADB**Em cada emulador aberto, acesse as configurações do sistema e certifique-se de que a opção Permissão Root (ou ADB Debugging) esteja habilitada. Isso é obrigatório para que o script envie os comandos de clique.

**3. Execução do Script**Inicie o arquivo executável (ou rode o arquivo Python no terminal).

**4. Reconhecimento Automático**O script fará uma varredura nas portas de rede locais. Ele identificará os emuladores ativos, lerá o nome de cada janela diretamente do Windows e exibirá o status individual no terminal. Cada emulador roda separadamente, sem interferir um com o outro.
